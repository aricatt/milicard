# 模块：主播/仓管列表

- **参考截图**: `3_主播_仓管.png`
- **URL**: `/live-base/base-data/personnel`

---

## 1. 功能概述

此页面用于管理系统中的“主播”和“仓管”人员。原则上，即使仓库没有固定人员管理，也需要设立一个虚拟的“仓库管理员”角色，用于库存管理和调货操作。

---

## 2. 页面布局与功能

### 2.1 页面选项卡

- **主播/仓管**: 默认激活的选项卡，显示下方定义的人员列表。

### 2.2 主要操作

- **添加**: 
  - 页面左上方应有一个“添加” (Add) 按钮。
  - 点击后，跳转到“添加人员”页面。

### 2.3 人员列表

- **数据展示**: 页面主体为一个表格，用于展示所有主播和仓管人员的信息。
- **分页**: 表格下方应有分页组件。
- **表格列**:
  - **ID**: 唯一标识符。
  - **编号**: 业务编号 (例如: `ANCHOR-9ZPSBQ99T7S`， 系统自动生成)。
  - **名称**: 人员的姓名或昵称。
  - **创建时间**: 记录的创建日期和时间。
  - **操作**:
    - **编辑 (Edit)**: 点击后，跳转到对应的编辑页面。
    - **删除 (Del)**: 点击后，应弹出确认对话框，确认后删除该记录。

---

## 3. 数据模型

```typescript
// file: src/models/personnel.ts

export enum PersonnelRole {
  Anchor = '主播',
  WarehouseKeeper = '仓管',
}

export interface Personnel {
  id: number;
  code: string;
  name: string;
  role: PersonnelRole; // 假设存在该字段用于区分角色
  createdAt: string; // ISO 8601 format
}
```

---

## 4. 已确认的业务规则

1.  **角色区分**: 根据 `readme` 文档，原系统通过名称来区分“主播”和“仓管”。
    - **新系统设计**: 为提高系统健壮性，我们将在数据模型中保留 `role` 字段。在“添加/编辑”页面，应提供一个下拉框或单选按钮，让用户明确选择人员的角色（主播或仓管）。通过名称来区分的。
