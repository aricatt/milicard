# 模块：直播间/仓库列表

- **参考截图**: `2_直播间_仓库.png`
- **URL**: `/live-base/base-data/live-rooms`

---

## 1. 功能概述

此页面用于统一管理系统中的“直播间”和“仓库”。

**核心业务逻辑**: “仓库”是采购商品后的总库存管理中心，而“直播间”是商品消耗和销售的前端单位。标准的库存流转路径是：**采购 → 到货至仓库 → 从仓库调货至直播间**。因此，在创建时明确区分“仓库”和“直播间”类型至关重要。

---

## 2. 页面布局与功能


### 2.2 主要操作

- **添加**: 
  - 页面左上方应有一个“添加” (Add) 按钮。
  - 点击后，跳转到“添加直播间/仓库”页面。

### 2.3 列表

- **数据展示**: 页面主体为一个表格，用于展示所有直播间和仓库信息。
- **分页**: 表格下方应有分页组件。
- **表格列**:
  - **ID**: 唯一标识符。
  - **编号**: 业务编号 (例如: `LIVE-KC9J5R13RSM`)。
  - **名称**: 直播间或仓库的名称。
  - **类型**: 区分是“直播间”还是“仓库”。
  - **创建时间**: 记录的创建日期和时间。
  - **操作**:
    - **编辑 (Edit)**: 点击后，跳转到对应的编辑页面。
    - **删除 (Del)**: 点击后，应弹出确认对话框，确认后删除该记录。

## 3. 数据模型

```typescript
// file: src/models/liveRoom.ts

export enum LiveRoomType {
  LiveRoom = '直播间',
  Warehouse = '仓库',
}

export interface LiveRoom {
  id: number;
  code: string;
  name: string;
  type: LiveRoomType;
  createdAt: string; // ISO 8601 format
}
```
